# Prime Quartet Discontinuity (Cross-Repo Boundary Record)

**Status:** Boundary record (documentation only)
**Date pinned:** 2026-05-24

## 0) Prime quartet vocabulary

Across the Interdependency ecosystem, the "prime quartet" is:

1. `ucns` (UCNS-A recursive factorization algebra)
2. `edcmbone` (structural/EDCM measurement package)
3. `a0` (runtime consumer/service layer)
4. `interdependent-lib` (meta-package/index surface)

## 1) Discontinuity statement

The quartet is intentionally interoperable but **not theorem-continuous**.

- UCNS-A theorem/proof status in `ucns` does not automatically propagate to
  `edcmbone` metrics, `a0` runtime outputs, or `interdependent-lib` package
  inclusion.
- UCNS-G/EDCM geometry and metric claims remain a parallel layer unless an
  explicit, source-backed bridge is implemented and verified.

This is a documented discontinuity boundary, not a temporary omission.

## 2) What continuity would require

Before any continuity claim is made across the quartet, the bridge artifacts in
`docs/edcm-edcmbone-bridge-checklist.md` must exist with source-backed tests,
including projection mapping, recoverability limits, per-output status labels,
and comparative fixture tests.

## 3) Repo-local commitments in `ucns`

Within this repository:

- `ucns-spec.md` and `docs/pure-ucns-number-system.md` are scoped to UCNS-A
  theorem/algebra status.
- `docs/ucns-shape-reconciliation.md` and
  `docs/ucns-g-prime-cylinder-supplement.md` preserve the PARALLEL verdict.
- Documentation must not claim that Theorem N already proves EDCM/UCNS-G.
