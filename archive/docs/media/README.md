# docs/media — provenance and claim policy

Images in this repository are divided into two classes, and the class
determines what they are allowed to assert.

## Claim-bearing figures (deterministic)

Authored as SVG with every string written and checked against the
claims ledger, then rendered to PNG. These may state results, statuses,
and formulas; they are held to the same drift discipline as prose.

| File | Content | Source of truth |
|---|---|---|
| `law-skeleton-figure.svg` / `.png` | Carrier-LCM Law Lean discharge state: composition checked, 4 theorems discharged (axiom audit: `propext` only), 5 leaves open | `formal/Ucns/CarrierLcm.lean`, `docs/carrier-support-pruning.md` |

If the discharge state changes, this figure is STALE until regenerated.
Stale claim-bearing figures should be regenerated or removed, not
captioned around.

## Decorative images (AI image-model output)

Generated by image models, selected for accuracy-of-mood rather than
accuracy-of-claim, and screened to assert nothing falsifiable. Any
numbers, labels, or glyphs visible in them are illustrative texture,
not data. They MUST NOT be used where a reader could mistake their
contents for results.

| File | Use |
|---|---|
| `ucns-emblem.jpg` / `ucns-emblem-portrait.jpg` | Repository / article hero image (Möbius sphere motif) |
| `theorem-n-factorization.jpg` | Illustration alongside Theorem N discussion (A, B composing to P) |
| `slice-embedding-schematic.jpg` | Schematic alongside the Carrier-LCM Law proof; the embedded caption ("pure slices embed both factors unchanged...") is correct, the grid cell values are illustrative only |

## Policy

Decoration comes from the sampler; statements come from the typesetter.
Image models in current use confidently render false specifics
(wrong operators, premature version numbers, checkmarks on open
obligations) — several candidate images were rejected for exactly this.
Text inside generated images is treated as untrusted by default.
