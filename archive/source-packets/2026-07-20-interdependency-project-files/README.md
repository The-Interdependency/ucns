# Reviewed source packet — 2026-07-20

## Purpose

This directory records the review and placement decision for the uploaded
`interdependency_project_files.zip` packet. The packet contained 69 top-level
files spanning UCNS, EDCM, A0, The Interdependent Way, speculative research,
and private business records.

This is a provenance record, not a restoration of pre-reset UCNS.

## Governing boundary

Current root canon remains [`CANON.md`](../../../CANON.md): every UCNS object is
a recursive Möbius carrier; its unique hidden twist/seam is zero; one visible
circuit reverses orientation; two circuits complete the return.

No uploaded implementation satisfies that object boundary. The recovered UCNS
lineage instead repeatedly:

- chooses the first anchor as zero during normalization;
- identifies `None` or a trivial anchor object with the multiplicative unit;
- computes host geometry on a one-circle modulus;
- stores face state independently of lawful seam-crossing parity;
- allows recursive payload absence rather than requiring a complete
  twist-bearing payload object; and
- states factorization or completeness results over that incomplete carrier.

Those are historical facts about the development lineage. They are not defects
to conceal, but they make the material ineligible for root promotion.

## Review result

### Preserved through the existing archive and Git history

The packet's useful UCNS material is historical evidence for:

- the flat kernel and early epicyclic object models;
- recursive quotient experiments;
- Class III failure-boundary discovery;
- witness-matrix and coupled-payload solver development;
- depth-2 oracle experiments;
- carrier-widening experiments; and
- the sequence of claims that eventually exposed the missing object invariant.

The current [`archive/`](../../) already preserves the later implementation,
specifications, tests, formal work, review packets, and experimental scripts.
The complete Git history preserves earlier versions. Duplicating those files
under new names would create false parallel authority and make provenance less
clear, so this packet is represented by the manifest and mapping in this
directory rather than by another code fork.

### Not promoted

None of the following enters the root:

- `UCNSObject` implementations from the packet;
- `normalize()` behavior that shifts the first anchor to zero;
- `None`-as-unit semantics;
- one-circle / 360-degree completion assumptions;
- pre-reset multiplication, quotient, factor-search, codec, embedding, or
  theorem claims; or
- tests whose success depends on those assumptions.

A passing historical test is evidence that an old implementation matched its
old specification. It is not evidence that the implementation satisfies
current UCNS canon.

### Excluded from this repository

The review excluded:

- California LLC filings, receipts, addresses, account fragments, and welcome
  letters;
- The Interdependent Way text, whose canon belongs in `wayseer00` only;
- A0 bootstrap material;
- EDCM source material, which is reviewed in `The-Interdependency/edcm`;
- the consciousness-primes prediction paper, which requires its own scientific
  and provenance review before any UCNS placement; and
- an empty lexicon placeholder.

## Reuse rule

A historical algorithm may be recovered only after it is restated as an
explicit projection of the complete twist-bearing object. Recovery must define:

1. what information the projection forgets;
2. how the hidden seam remains intrinsic and immovable;
3. how 720-degree return and orientation state are represented;
4. how recursive payloads remain complete UCNS objects;
5. which old tests remain meaningful under the projection; and
6. which old theorem statements must be weakened, replaced, or discarded.

Until those obligations are met, historical code remains evidence only.
